## What I built this sprint (and why these earned the spot)
- I have built User registration/login,-User authentication is important because it creates accountability and trust between borrowers and owners. It also ensures that only registered users can make bookings, helping to protect user information and making the marketplace more secure.
Home page showing nearby items,The home page is the user's first interaction with the platform. Displaying nearby items immediately showcases what is available in the user's neighbourhood, making it easy to browse and encouraging users to start exploring items they can borrow.
Search and filtering,As the number of listings grows, users need an efficient way to find what they are looking for. Search and filters by category, distance and free/paid options improve usability and help users locate relevant items quickly, creating a better overall experience.
Item details,Users need enough information before deciding to borrow an item. The item details page provides photos, descriptions, owner information, pricing, availability, and booking options, allowing users to make informed decisions.
Booking system,The booking system is the core functionality of the marketplace. It allows users to select borrowing dates, confirm reservations, and complete the borrowing process. Without this feature, users would only be able to browse listings without actually using the platform.

## What I cut or deferred (and why)
- I have removed the countdown timer.The feature displaying messages such as "3 people are viewing this item right now" has been removed. It does not improve the core functionality and may create misleading urgency if the data is not genuine.

-Offline functionality has been removed from the MVP due to its complexity. Since listings, bookings, and availability change frequently, an internet connection is required to ensure users always see accurate information.

-Referral and invite systems have been excluded from the MVP. Growth features are more valuable after the product has been validated and users are actively engaging with the platform.

-I also removed the wishlist.Users will not be able to save items to a wishlist in the MVP. This feature does not contribute to the core borrowing flow and can be added once the platform has an active user base.
## What I pushed back on (and why — be honest and kind)
- The original idea was to require users to sign up before they could view any listings. I recommended allowing users to browse items first and only requiring registration when they want to make a booking. This reduces friction for new users and lets them see the value of the platform before creating an account.

Offline functionality was postponed because the application relies on current information such as item availability and bookings. Supporting offline access would significantly increase development complexity while providing limited value for the initial release.
